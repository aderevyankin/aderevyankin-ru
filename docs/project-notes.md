# Заметки по проекту

## Цели
- Показать, как сделать сайт максимально простым, но при этом работоспособным, масштабируемым, дешевым в поддержке и устойчивым.
- Дать прозрачный пример продакшн-реализации без лишней сложности.

## Не-цели
- Не универсальный шаблон под любой бизнес.
- Не витрина анимаций и «фронтенд‑фокусов».
- Не полноценная CMS или платформа с кабинетом.

## Нефункциональные требования
- **Скорость:** минимальный JS, статический экспорт, быстрый рендер.
- **Доступность:** семантика, клавиатурная навигация, контраст.
- **Стабильность:** zero layout shift, предсказуемая верстка.
- **SEO:** метаданные, чистая структура, Open Graph.

## Архитектурные решения
- **Next.js (static export):** статическая сборка, дешевый хостинг, высокая надежность.
- **App Router:** понятная файловая структура, разделение layout и страницы.
- **CSS Modules:** изоляция стилей без CSS‑in‑JS и рантайма.
- **Yandex Object Storage:** простой статический хостинг без серверов.
- **Yandex Cloud Functions + Telegram:** форма без бекенда и базы, быстрые уведомления.

## Компромиссы и ограничения
- Одна страница, без динамического контента и CMS.
- Минимум интерактивности ради скорости и устойчивости.
- Нет хранения заявок — сообщения уходят сразу в Telegram.
